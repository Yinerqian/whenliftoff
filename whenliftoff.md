# whenliftoff 产品需求文档 PRD

## 1. 产品概述

**产品名称：** whenliftoff
**产品形态：** 响应式网页版项目，适配 PC 浏览器和手机浏览器
**产品定位：** 面向中文用户的全球火箭发射日程与航天新闻聚合网站
**一句话介绍：** 用中文、清晰、美观的方式，告诉用户全球火箭什么时候发射、在哪里发射、执行什么任务，以及最近发生了哪些航天新闻。

whenliftoff 主要通过 Launch Library 2 获取全球火箭发射、任务、航天机构、发射场等数据；通过 Spaceflight News API 获取航天新闻、博客和报告数据。Launch Library 2 是 The Space Devs 提供的发射与航天事件 API，覆盖轨道发射和部分热门亚轨道发射，并提供 launches、events、agencies、locations、pads 等数据类型；免费访问存在请求频率限制，因此项目需要做后端缓存和定时同步。Spaceflight News API 提供新闻文章、博客和报告，并且其新闻数据可以关联 Launch Library 2 的发射或事件 ID，适合做“某次发射相关新闻”的功能。

## 2. 项目背景

目前中文互联网上关于全球火箭发射的信息分散、更新不稳定，很多原始数据是英文，普通用户需要自己理解发射任务、时间、地点、任务状态和新闻背景。whenliftoff 希望把这些公开航天数据进行中文化、结构化和展示，让用户可以快速了解：

1. 最近有哪些火箭即将发射；
2. 某次发射的倒计时、时间、地点、火箭、任务和发射状态；
3. 最近有哪些重要航天新闻；
4. 某次发射和相关新闻之间有什么关联；
5. 在 PC 和手机上都能获得良好的浏览体验。

## 3. 目标用户

### 3.1 核心用户

1. **航天爱好者**
   关注 SpaceX（太空探索技术公司）、NASA（美国国家航空航天局）、ESA（欧洲航天局）、CNSA（中国国家航天局）、Rocket Lab（火箭实验室）等机构的发射信息，希望快速查看全球发射日程。

2. **泛科技用户**
   平时不深入研究航天，但对火箭发射、太空探索、卫星、载人航天新闻感兴趣。

3. **内容创作者 / 科普博主**
   需要快速获取发射时间、任务简介、相关新闻，用于写文章、做视频或发布社交媒体内容。

4. **学生 / 科普学习用户**
   希望通过中文内容了解全球航天动态。

## 4. 产品目标

### 4.1 MVP 目标

第一阶段先做一个可上线、可浏览、可持续更新的响应式网站，重点完成：

1. 展示全球即将发射的火箭列表；
2. 展示每次发射的详情页；
3. 展示航天新闻列表和新闻详情；
4. 支持中文化展示核心字段（机构名称同时展示英文与中文，英文更突出）；
5. 支持 PC 和手机自适应；
6. 支持基础 SEO；
7. 支持数据缓存，避免前端直接高频请求第三方 API。

### 4.2 长期目标

后续逐步增加：

1. 发射提醒；
2. 按国家、机构、火箭、发射场筛选；
3. 发射日历；
4. 热门任务专题页；
5. AI 航天新闻摘要；
6. 用户收藏；
7. 中英文切换；
8. 历史发射统计；
9. 发射直播入口聚合；
10. 微信/Telegram/邮件提醒。

## 6. 中文化规则

## 6.1 发射状态中文化

| 原状态             | 中文展示  |
| --------------- | ----- |
| Go              | 计划发射  |
| TBD             | 时间待定  |
| TBC             | 时间待确认 |
| Success         | 发射成功  |
| Failure         | 发射失败  |
| Partial Failure | 部分失败  |
| Hold            | 暂停    |
| In Flight       | 飞行中   |

## 6.2 时间中文化

所有发射时间至少展示三种格式：

1. 北京时间：2026 年 7 月 10 日 21:30
2. 用户本地时间：根据浏览器时区自动转换
3. UTC 时间：2026-07-10 13:30 UTC

如果发射时间为 TBD/TBC，应展示：

> 发射时间尚未最终确认，后续可能调整。

## 6.3 机构名称展示规则（中英文同时显示）

机构名称必须同时展示英文与中文，其中英文为主展示，中文作为辅助说明：

| 英文（主展示）     | 中文（辅助说明）  |
| ----------- | --------- |
| SpaceX      | 太空探索技术公司  |
| NASA        | 美国国家航空航天局 |
| ESA         | 欧洲航天局     |
| Rocket Lab  | 火箭实验室     |
| Blue Origin | 蓝色起源      |
| Roscosmos   | 俄罗斯国家航天集团 |

展示规范：

* 列表页：SpaceX（太空探索技术公司）
* 详情页标题区域：**SpaceX**（太空探索技术公司）
* 移动端：优先显示英文，中文可换行或缩小字号显示

## 6.4 任务摘要中文化

任务简介不应直接机器翻译，而应改写为中文用户容易理解的表达：

英文原始内容：
“Falcon 9 will launch a batch of Starlink satellites into low Earth orbit.”

中文展示：
“猎鹰 9 号火箭将把一批 Starlink 卫星送入近地轨道，用于扩展 SpaceX 的卫星互联网服务。”

## 8. 技术方案建议

## 8.1 推荐架构

前端：

* Next.js / React
* Tailwind CSS
* 响应式布局
* 部署到 Vercel

后端 / 数据：

* Supabase PostgreSQL
* Supabase Edge Functions 或 Vercel Serverless Functions
* 定时任务同步 Launch Library 2 和 Spaceflight News API 数据
* 数据入库后再由前端读取

## 8.2 为什么不建议前端直接请求第三方 API

Launch Library 2 免费访问存在请求频率限制，因此 whenliftoff 应采用“后端定时同步 + 数据库缓存 + 前端读取本站接口”的方式，避免每个用户访问页面时直接请求第三方 API。

推荐流程：

1. 定时任务每 1–6 小时同步发射数据；
2. 新闻数据每 30–60 分钟同步一次；
3. 同步后写入数据库；
4. 前端页面读取本站 API；
5. 用户访问量增加后，可增加缓存层（如 Redis/CDN）。
